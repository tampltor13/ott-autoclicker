[PREVIEW] v1.0.34 — GPU flagovi za Chrome (reverted)
------------------------------------------------------
Cilj: popraviti crni ekran u World2Capture region/window capture.
Isprobano redom:
- --disable-gpu, --disable-gpu-compositing, --disable-accelerated-2d-canvas,
  --disable-accelerated-video-decode (od v1.0.23, ostalo aktivno)
- --use-gl=swiftshader (potpuni software renderer, zaobilazi GPU/ANGLE/D3D)
- --use-angle=swiftshader (forsira ANGLE software path)
- --disable-features=VizDisplayCompositor
Zaključak: nijedan flag nije pomogao. Problem nije u GPU flagovima nego u
načinu na koji Selenium pokreće Chrome (--remote-debugging-port CDP konekcija
mijenja DWM kompoziciju prozora). Reverted.


[PREVIEW] v1.0.35 — Chrome subprocess launch (reverted)
---------------------------------------------------------
Cilj: pokrenuti Chrome kao normalan OS subprocess (ne preko Selenium
webdriver.Chrome()) pa da World2Capture može čitati prozor.
Implementirano:
- subprocess.Popen(chrome.exe --remote-debugging-port=9222 ...)
- čeka 3s → Selenium se spaja via debuggerAddress CDP
- Connect button za spajanje na već-otvoreni Chrome
- chrome_proc cleanup u close_browser()
Potvrđeno: --remote-debugging-port sam po sebi NE kvari World2Capture
capture (testom ručnog pokretanja Chrome s flagom capture je radio).
Problem: kompleksnost spajanja, Chrome ignorira flag ako je već pokrenut,
korisnik mora killati sve chrome.exe procese. Reverted, nastavlja se kasnije.
